# Git 工作流程笔记

## 场景：新入职实习生完成功能开发并提交 PR

### 1. 克隆公司代码仓库到本地
- **命令**：`git clone <仓库地址>`
- **目的**：将远程仓库的代码复制到本地，作为起点。

### 2. 创建一个新的功能分支
- **命令**：`git checkout -b feature-A`
- **目的**：在本地创建一个名为 `feature-A` 的新分支，用于开发新的功能，避免直接在主分支上工作。

### 3. 开发功能并提交本地更改
- **命令**：`git add .` 然后 `git commit -m "Add feature A"`
- **目的**：将完成的功能代码变更提交到本地仓库的 `feature-A` 分支上。

### 4. 将功能分支推送到远程仓库
- **命令**：`git push -u origin feature-A`
- **目的**：将本地的 `feature-A` 分支推送到远程仓库，并设置上游分支。

### 5. 在 GitHub 上创建 Pull Request (PR)
- **步骤**：登录 GitHub，进入仓库页面，点击 “New Pull Request” 按钮，选择 `feature-A` 分支为来源分支，将其与主分支 (比如 `master`) 进行对比。

- **目的**：创建一个 PR，请求将 `feature-A` 分支的改动合并到主分支，并让团队成员审核。

### 6. 审核与合并 PR
- **说明**：在 PR 创建后，团队的代码审查者会查看你的改动，评论反馈，并在确认无误后合并 PR，将你的代码正式合并到主分支。
